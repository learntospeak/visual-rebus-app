# Visual Rebus App — Start Here

This folder is a no-code planning shell for building the app in VS Code.

## Immediate objective

Prove the core experience with **25 excellent puzzles** before adding ads, accounts, coins, multiplayer or a large content library.

## Open these files in order

1. `Visual_Rebus_App_Project_Plan.pdf` — market, product, design, timing and launch strategy.
2. `docs/01_BUILD_CHECKLIST.md` — exact build sequence and estimated effort.
3. `docs/02_PUZZLE_CONTENT_SPEC.md` — required fields and quality rules for every puzzle.
4. `docs/05_GUI_STYLE_GUIDE.md` — visual direction and screen rules.
5. `docs/03_LAUNCH_CHECKLIST.md` — testing, privacy, monetisation and store work.
6. `docs/04_MARKET_RESEARCH_NOTES.md` — competitor weaknesses and positioning.

## First 48 hours

### Block 1 — Scope and content: 3–4 hours

- Choose a temporary working name.
- Write 10 puzzle specifications across at least four formats.
- Reject any puzzle whose answer or explanation is debatable.
- Decide one product promise: **fair visual riddles with useful clues and clear explanations**.

### Block 2 — Core screens: 4–6 hours

- Build only: Home → Puzzle → Solved → Next.
- Use the supplied style guide and mock-up.
- Keep coins, streaks and menus visually secondary.

### Block 3 — Puzzle loop: 5–8 hours

- Display a puzzle.
- Show answer word lengths.
- Accept spelling and punctuation variants.
- Supply three progressive clues.
- Explain every visual element after the solve or reveal.
- Save progress locally.

### Block 4 — Prototype content and testing: 4–6 hours

- Expand to 25 polished puzzles.
- Test with at least three people who did not help create them.
- Record every confusing answer or failed explanation.
- Remove weak puzzles rather than keeping them to inflate the count.

## Continue only when

- New testers understand the game without lengthy instructions.
- At least 2 of 3 testers can solve most easy puzzles before the final clue.
- Players describe the reveal as satisfying rather than arbitrary.
- Progress, answer validation and the next-puzzle flow work reliably.

## Launch scope

- **120 playable puzzles** at launch.
- **30 reserve puzzles** for daily content and rapid replacements.
- 40 easy, 50 medium and 30 hard.
- At least six visual formats.
- Android-first testing; web version can run while store testing is underway.

## Important boundary

Do not copy another app's name, icon, screenshots, source code, exact interface or puzzle collection. Use common phrases only as answer concepts, then create original artwork, layouts, clues and explanations.

No application code is included in this folder.
